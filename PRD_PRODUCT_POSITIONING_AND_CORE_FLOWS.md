# 「皖美」安徽文旅小程序 - 产品需求文档（PRD）

## 一、产品定位

### 1.1 产品概述

**产品名称**：皖美（Wanmei）

**产品类型**：文旅展示类微信小程序

**产品定位**：一款以徽派美学为设计语言，集内容展示、互动问答、社区参与于一体的安徽文旅数字名片。通过沉浸式的视觉体验和趣味化的互动方式，向用户全方位展示安徽的美食文化、自然风景和历史文化底蕴。

### 1.2 目标用户

**主要用户群体**：
- **文旅爱好者**（占比40%）：对安徽文化、旅游景点感兴趣的用户
- **本地居民**（占比30%）：希望深入了解家乡文化的安徽本地用户
- **潜在游客**（占比20%）：计划前往安徽旅游的外地用户
- **文化学习者**（占比10%）：对徽文化、传统文化感兴趣的学习者

**用户画像**：
- **年龄分布**：18-45岁为主，其中25-35岁占比最高
- **地域分布**：安徽本地用户占40%，长三角地区用户占30%，其他地区占30%
- **使用场景**：
  - 旅游规划前了解安徽景点和美食
  - 闲暇时间浏览文化内容
  - 社交分享安徽特色内容
  - 学习徽文化知识

### 1.3 产品价值

**对用户的价值**：
- **信息获取**：一站式了解安徽美食、风景、文化的权威平台
- **文化传承**：通过趣味化方式学习和传播徽文化
- **互动体验**：点赞、收藏、评论、答题等多维度互动
- **个性化服务**：根据兴趣收藏内容，记录学习轨迹

**对平台的价值**：
- **文化传播**：数字化展示安徽文化，提升安徽文旅品牌影响力
- **用户沉淀**：通过内容和互动留存用户，建立文旅社区
- **数据积累**：收集用户偏好数据，为精准营销提供支持
- **内容共创**：全员可编辑的内容管理机制，降低运营成本

### 1.4 产品特色

**核心差异化优势**：

1. **徽派美学设计**
   - 采用安徽山水蓝（#4A90E2）和古徽州墨绿（#2E4B3F）配色
   - 融合徽派建筑元素（马头墙、窗棂纹理）
   - 印章风格图标和渐变效果

2. **趣味化学习**
   - 徽文化小问答，寓教于乐
   - 答对收集印章，增强成就感
   - 随机题目机制，保持新鲜感

3. **全员可编辑**
   - 无需后台管理系统，小程序内直接编辑
   - 降低内容更新门槛，提高更新频率
   - 支持题库、景点、美食、文化四大模块管理

4. **轻量化交互**
   - 无需注册登录，UUID标识用户
   - 点赞、收藏、评论一键操作
   - 流畅的页面切换和动画效果

### 1.5 产品目标

**短期目标（3个月）**：
- 用户量达到 5,000+
- 日活跃用户（DAU）达到 500+
- 内容浏览量达到 50,000+
- 问答参与率达到 30%

**中期目标（6个月）**：
- 用户量达到 20,000+
- 日活跃用户（DAU）达到 2,000+
- 用户平均停留时长达到 5 分钟
- 内容收藏率达到 15%

**长期目标（1年）**：
- 成为安徽文旅领域头部小程序
- 建立活跃的文旅社区
- 与安徽文旅局、景区建立合作
- 探索商业化变现路径

## 二、核心功能流程

### 2.1 文旅内容浏览流程

#### 2.1.1 功能概述

用户可以通过多种方式浏览安徽的美食、风景、文化内容，支持分类筛选、详情查看、点赞收藏评论等互动操作。

#### 2.1.2 流程图

```
┌─────────────────────────────────────────────────────────────┐
│                      文旅内容浏览流程                          │
└─────────────────────────────────────────────────────────────┘

[用户进入小程序]

│   首页展示         │

│ • 分类入口卡片     │

└───────────────────┘
        ↓
    用户选择
        ↓
   ┌────┴────┐
   ↓         ↓
[点击分类]  [点击内容卡片]

#### 2.1.3 详细步骤

**步骤1：进入首页**
- **触发条件**：用户打开小程序或点击底部TabBar"首页"
- **页面元素**：
  - 顶部沉浸式Banner（占屏幕40%高度）
  - 4个分类入口卡片（徽菜美食、自然风景、历史文化、徽文化小问答）
  - 推荐内容列表（按分类分组展示）
- **交互说明**：
  - Banner自动轮播展示精选内容
  - 分类卡片点击跳转到对应分类页
  - 内容卡片点击跳转到详情页
  - 下拉刷新加载最新内容

**步骤2：选择分类浏览**
- **触发条件**：用户点击分类入口卡片或底部TabBar分类标签
- **页面元素**：
  - 徽派风格顶部Banner（展示分类主题）
  - 内容列表（瀑布流或列表布局）
  - 每个内容卡片包含：图片、标题、副标题、点赞数
- **交互说明**：
  - 支持下拉刷新
  - 支持上拉加载更多
  - 卡片点击跳转到详情页
  - 卡片有悬浮缩放动画效果

**步骤3：查看内容详情**
- **触发条件**：用户点击内容卡片
- **页面元素**：
  - 顶部大图（全屏宽度）
  - 标题和副标题
  - 详细描述文本
  - 点赞/收藏按钮（底部固定）
  - 评论列表
  - 评论输入框
- **交互说明**：
  - 图片支持点击放大查看
  - 点赞按钮点击后变红色，再次点击取消
  - 收藏按钮点击后变实心，再次点击取消
  - 评论输入框点击弹出键盘
  - 评论提交后立即显示在列表顶部

**步骤4：点赞操作**
- **触发条件**：用户点击点赞按钮
- **业务逻辑**：
  1. 检查用户是否已点赞
  2. 如未点赞：插入点赞记录 + 点赞数+1
  3. 如已点赞：删除点赞记录 + 点赞数-1
  4. 更新UI显示
- **视觉反馈**：
  - 点赞按钮颜色变化（灰色 ↔ 红色）
  - 点赞数字实时更新
  - 按钮有缩放动画效果

**步骤5：收藏操作**
- **触发条件**：用户点击收藏按钮
- **业务逻辑**：
  1. 检查用户是否已收藏
  2. 如未收藏：插入收藏记录 + 收藏数+1
  3. 如已收藏：删除收藏记录 + 收藏数-1
  4. 更新UI显示
- **视觉反馈**：
  - 收藏按钮图标变化（空心 ↔ 实心）
  - 收藏数字实时更新
  - 显示Toast提示"已收藏"或"已取消收藏"

**步骤6：发表评论**
- **触发条件**：用户点击评论输入框并输入内容
- **业务逻辑**：
  1. 用户输入评论内容
  2. 点击发送按钮
  3. 校验内容不为空
  4. 插入评论记录（包含用户ID、昵称、内容、时间）
  5. 评论列表顶部插入新评论
- **视觉反馈**：
  - 输入框获得焦点时高亮
  - 发送按钮在有内容时高亮
  - 提交成功显示Toast"评论成功"
  - 评论列表自动滚动到顶部

**步骤7：查看我的收藏**
- **触发条件**：用户点击底部TabBar"我的"，进入个人中心
- **页面元素**：
  - 用户信息区（昵称、印章数量）
  - 我的收藏列表
  - 收集的印章展示
- **交互说明**：
  - 收藏列表展示所有已收藏的内容
  - 点击收藏项跳转到详情页
  - 支持取消收藏操作

#### 2.1.4 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 网络请求失败 | 显示Toast"加载失败，请稍后重试"，提供重试按钮 |
| 内容为空 | 显示空状态提示"暂无内容" |
| 评论内容为空 | 显示Toast"评论内容不能为空" |
| 评论内容过长 | 限制最大字符数（200字），超出时禁用发送按钮 |
| 图片加载失败 | 显示占位图，提供重新加载选项 |

### 2.2 徽派文化趣味问答流程

#### 2.2.1 功能概述

用户通过答题的方式学习徽文化知识，答对题目可以收集印章，增强学习的趣味性和成就感。采用随机题目机制，每次答题都有新鲜感。

#### 2.2.2 流程图

```
┌─────────────────────────────────────────────────────────────┐
│                   徽派文化趣味问答流程                          │
└─────────────────────────────────────────────────────────────┘

[用户进入问答页]
        ↓
┌───────────────────┐
│  问答页初始化      │
│ • 随机加载题目     │
│ • 显示题干         │
│ • 显示3个选项      │
└───────────────────┘
        ↓
    [用户答题]
        ↓
   ┌────┴────┐
   ↓         ↓
[选择选项] [点击提交]
   ↓         ↓
[选项高亮] [校验答案]
   ↓         ↓
   └────┬────┘
        ↓
    判断结果
        ↓
   ┌────┴────┐
   ↓         ↓
[答对]    [答错]
   ↓         ↓
┌──────┐ ┌──────┐
│显示✓  │ │显示✗  │
│收集印章│ │显示解析│

└──────┘ └──────┘
   ↓         ↓
   └────┬────┘
        ↓
  [显示解析内容]
        ↓
  [下一题按钮]
        ↓
    用户选择
        ↓
   ┌────┴────┐
   ↓         ↓
[继续答题] [返回首页]
   ↓         ↓
[加载新题目] [保存进度]
   ↓         ↓
[重复流程]  [结束]
```

#### 2.2.3 详细步骤

**步骤1：进入问答页**
- **触发条件**：
  - 用户点击首页"徽文化小问答"分类卡片
  - 或通过其他入口进入问答页
- **页面元素**：
  - 徽派风格顶部Banner（展示"徽文化小问答"标题）
  - 印章收集进度展示
  - 开始答题按钮
- **交互说明**：
  - 显示已收集的印章数量
  - 点击"开始答题"按钮进入答题界面

**步骤2：加载题目**
- **触发条件**：用户点击"开始答题"或"下一题"
- **业务逻辑**：
  1. 从题库中随机抽取一道题目
  2. 确保不重复最近答过的题目（如果题库足够大）
  3. 加载题目内容：题干、选项A/B/C、正确答案、解析
- **页面元素**：
  - 题目序号（如"第1题"）
  - 题干文本
  - 3个选项按钮（A/B/C）
  - 提交按钮（初始禁用）
- **视觉效果**：
  - 题目以淡入动画出现
  - 选项按钮采用徽派配色

**步骤3：选择答案**
- **触发条件**：用户点击选项按钮
- **交互说明**：
  - 点击选项后，该选项高亮显示（实心背景+白色文字）
  - 其他选项恢复默认样式
  - 提交按钮变为可用状态（高亮）
  - 支持重新选择，点击其他选项可切换
- **视觉反馈**：
  - 选中的选项有明显的视觉区分
  - 选项按钮有点击缩放动画

**步骤4：提交答案**
- **触发条件**：用户点击"提交"按钮
- **业务逻辑**：
  1. 获取用户选择的答案
  2. 与正确答案进行比对
  3. 判断答对或答错
  4. 如果答对：
     - 生成印章（基于题目ID或时间戳）
     - 保存印章到本地存储
     - 印章数量+1
  5. 显示答题结果和解析
- **视觉反馈**：
  - 提交按钮显示加载状态（防止重复提交）
  - 页面禁用所有选项按钮

**步骤5：显示答题结果**
- **触发条件**：答案提交后
- **答对的情况**：
  - 显示大大的绿色"✓"图标
  - 显示"回答正确！"文字
  - 显示印章收集动画（印章从屏幕中心飞向顶部）
  - 显示题目解析
  - 印章数量+1，更新顶部显示
- **答错的情况**：
  - 显示红色"✗"图标
  - 显示"回答错误"文字
  - 显示正确答案（高亮正确选项）
  - 显示题目解析
  - 显示鼓励语句"继续加油！"
- **页面元素**：
  - 结果图标（✓或✗）
  - 结果文字
  - 正确答案标注
  - 解析内容（详细说明）
  - "下一题"按钮
  - "返回首页"按钮

**步骤6：查看解析**
- **触发条件**：答题结果显示后自动展示
- **页面元素**：
  - 解析标题"题目解析"
  - 解析内容（详细说明答案的背景知识）
  - 相关知识扩展（可选）
- **交互说明**：
  - 解析内容支持滚动查看
  - 解析区域采用卡片样式，与题目区分

**步骤7：继续答题或返回**
- **触发条件**：用户查看完解析后
- **用户选择**：
  - **点击"下一题"**：
    - 清空当前题目
    - 重新执行步骤2（加载新题目）
    - 重复答题流程
  - **点击"返回首页"**：
    - 保存答题进度（印章数量）
    - 返回首页
    - 可在"我的"页面查看收集的印章

**步骤8：查看印章收集**
- **触发条件**：用户进入"我的"页面
- **页面元素**：
  - 印章收集区域
  - 显示所有已收集的印章（网格布局）
  - 每个印章显示收集时间
  - 印章总数统计
- **交互说明**：
  - 印章采用徽派印章风格设计
  - 点击印章可查看对应题目信息（可选）

#### 2.2.4 题目规则

| 规则项 | 说明 |
|-------|------|
| 题目来源 | 从数据库quiz_questions表随机抽取 |
| 题目类型 | 单选题（3个选项：A/B/C） |
| 题目难度 | 简单到中等，适合大众用户 |
| 题目内容 | 涵盖安徽美食、风景、文化、历史等知识 |
| 答题次数 | 无限制，用户可以反复答题 |
| 印章收集 | 每答对一题收集一个印章，印章不重复 |
| 题目更新 | 管理员可在内容管理页面新增/编辑/删除题目 |

#### 2.2.5 印章机制

**印章生成规则**：
- 每答对一道题目，生成一个唯一印章
- 印章ID = 题目ID + 答题时间戳
- 印章存储在本地Storage中

**印章展示**：
- 印章采用徽派印章风格（红色方形，内含文字）
- 印章文字为题目关键词或序号
- 印章按收集时间倒序排列

**印章价值**：
- 成就感：收集印章是学习的激励机制
- 社交性：可分享印章收集进度（未来功能）
- 游戏化：增加答题的趣味性

#### 2.2.6 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 题库为空 | 显示"题库正在建设中，敬请期待" |
| 网络请求失败 | 显示Toast"加载失败，请稍后重试"，提供重试按钮 |
| 未选择答案就提交 | 提交按钮保持禁用状态，无法点击 |
| 印章保存失败 | 静默处理，不影响答题流程，下次答对时补发 |
| 重复答同一题 | 允许重复答题，但不重复收集印章 |

### 2.3 内容管理（可修改）流程

#### 2.3.1 功能概述

全员可在小程序内直接编辑内容，无需后台管理系统。支持题库、景点、美食、文化四大模块的新增、编辑、删除操作，降低内容更新门槛，提高更新频率。

#### 2.3.2 流程图

```
┌─────────────────────────────────────────────────────────────┐
│                      内容管理流程                              │
└─────────────────────────────────────────────────────────────┘

[用户进入"我的"页面]
        ↓
[点击"内容管理"按钮]
        ↓
┌───────────────────┐
│ 内容管理总页       │
│ • 题库管理         │
│ • 景点管理         │
│ • 美食管理         │
│ • 文化管理         │
└───────────────────┘
        ↓
    用户选择模块

#### 2.3.3 详细步骤

**步骤1：进入内容管理**
- **触发条件**：
  - 用户在"我的"页面点击"内容管理"按钮
- **权限说明**：
  - 全员可访问，无需特殊权限
  - 所有用户都可以编辑内容
- **页面元素**：
  - 徽州窗棂纹理背景
  - 4个管理模块入口卡片：
    - 题库管理（显示题目总数）
    - 景点管理（显示景点总数）
    - 美食管理（显示美食总数）
    - 文化管理（显示文化总数）
  - 返回按钮
- **交互说明**：
  - 点击模块卡片进入对应管理页面
  - 卡片显示当前模块的内容数量
  - 卡片有悬浮缩放动画

**步骤2：选择管理模块**

**2.1 题库管理**
- **触发条件**：用户点击"题库管理"卡片
- **页面元素**：
  - 题目列表（显示题干和正确答案）
  - 每个题目有"编辑"和"删除"按钮
  - 顶部"新增题目"按钮
  - 返回按钮
- **交互说明**：
  - 列表按创建时间倒序排列
  - 点击"编辑"进入题目编辑页
  - 点击"删除"弹出确认对话框
  - 点击"新增题目"进入题目编辑页（空表单）

**2.2 内容管理（景点/美食/文化）**
- **触发条件**：用户点击"景点管理"/"美食管理"/"文化管理"卡片
- **页面元素**：
  - 内容列表（显示标题、副标题、缩略图）
  - 每个内容有"编辑"按钮
  - 返回按钮
- **交互说明**：
  - 列表按创建时间倒序排列
  - 点击"编辑"进入内容编辑页
  - 点击内容卡片可预览详情

**步骤3：新增内容**

**3.1 新增题目**
- **触发条件**：用户点击"新增题目"按钮
- **页面元素**：
  - 题干输入框（多行文本）
  - 选项A输入框
  - 选项B输入框
  - 选项C输入框
  - 正确答案选择器（A/B/C单选）
  - 解析输入框（多行文本）
  - 提交按钮
  - 取消按钮
- **交互说明**：
  - 所有字段必填
  - 正确答案选择器采用实心背景+白色文字样式
  - 点击提交前进行表单校验
  - 提交成功后返回题目列表

**3.2 新增内容（景点/美食/文化）**
- **触发条件**：暂不支持新增，仅支持编辑现有内容
- **说明**：新增功能可在后续版本中开放

**步骤4：编辑内容**

**4.1 编辑题目**
- **触发条件**：用户点击题目的"编辑"按钮
- **页面元素**：
  - 与新增题目相同的表单
  - 表单预填充当前题目的数据
  - 提交按钮
  - 取消按钮
- **交互说明**：
  - 用户可修改任意字段
  - 点击提交前进行表单校验
  - 提交成功后返回题目列表
  - 列表中对应题目更新

**4.2 编辑内容（景点/美食/文化）**
- **触发条件**：用户点击内容的"编辑"按钮
- **页面元素**：
  - 标题输入框（预填充）
  - 副标题输入框（预填充）
  - 描述输入框（多行文本，预填充）
  - 图片预览（显示当前图片）
  - 图片URL输入框（预填充）
  - 提交按钮
  - 取消按钮
- **交互说明**：
  - 用户可修改任意字段
  - 图片URL修改后，预览图自动更新
  - 点击提交前进行表单校验
  - 提交成功后返回内容列表
  - 列表中对应内容更新

**步骤5：删除内容**

**5.1 删除题目**
- **触发条件**：用户点击题目的"删除"按钮
- **交互说明**：
  1. 弹出确认对话框："确定要删除这道题目吗？"
  2. 用户选择：
     - 点击"确定"：执行删除操作
     - 点击"取消"：关闭对话框，不执行删除
  3. 删除成功后：
     - 显示Toast"删除成功"
     - 题目从列表中移除
     - 列表自动刷新

**5.2 删除内容（景点/美食/文化）**
- **说明**：暂不支持删除内容，避免误操作导致数据丢失

**步骤6：数据校验**

**题目表单校验规则**：
| 字段 | 校验规则 | 错误提示 |
|------|---------|---------|
| 题干 | 不能为空，最少5个字符 | "题干不能为空且至少5个字符" |
| 选项A | 不能为空 | "选项A不能为空" |
| 选项B | 不能为空 | "选项B不能为空" |
| 选项C | 不能为空 | "选项C不能为空" |
| 正确答案 | 必须选择A/B/C之一 | "请选择正确答案" |
| 解析 | 不能为空，最少10个字符 | "解析不能为空且至少10个字符" |

**内容表单校验规则**：
| 字段 | 校验规则 | 错误提示 |
|------|---------|---------|
| 标题 | 不能为空，最少2个字符 | "标题不能为空且至少2个字符" |
| 副标题 | 不能为空，最少2个字符 | "副标题不能为空且至少2个字符" |
| 描述 | 不能为空，最少20个字符 | "描述不能为空且至少20个字符" |
| 图片URL | 不能为空，必须是有效URL | "请输入有效的图片URL" |

**步骤7：提交保存**
- **触发条件**：用户点击"提交"按钮
- **业务逻辑**：
  1. 前端表单校验
  2. 如果校验失败：
     - 显示错误提示Toast
     - 停留在编辑页面
     - 用户修改后可重新提交
  3. 如果校验通过：
     - 显示加载状态（防止重复提交）
     - 调用数据库API保存数据
     - 等待服务器响应
  4. 保存成功：
     - 显示Toast"保存成功"
     - 返回列表页面
     - 列表自动刷新，显示最新数据
  5. 保存失败：
     - 显示Toast"保存失败，请稍后重试"
     - 停留在编辑页面
     - 用户可重新提交

**步骤8：返回列表**
- **触发条件**：
  - 保存成功后自动返回
  - 用户点击"取消"按钮
  - 用户点击返回按钮
- **交互说明**：
  - 如果有未保存的修改，弹出确认对话框："有未保存的修改，确定要离开吗？"
  - 返回列表后，列表自动刷新显示最新数据

#### 2.3.4 权限说明

**当前版本**：
- 全员可编辑，无需登录或特殊权限
- 所有用户都可以新增、编辑、删除内容
- 适合小规模团队或信任度高的用户群体

**未来版本（可选）**：
- 引入管理员角色，只有管理员可编辑
- 普通用户可提交编辑建议，管理员审核后生效
- 记录编辑历史，支持版本回退

#### 2.3.5 数据安全

**防止误操作**：
- 删除操作需要二次确认
- 编辑页面离开前提示未保存的修改
- 重要字段（如分类）不可修改，避免数据混乱

**数据备份**：
- 数据库自动备份（Supabase提供）
- 支持导出数据（未来功能）

**内容审核**（未来功能）：
- 敏感词过滤
- 图片内容审核
- 用户举报机制

#### 2.3.6 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 网络请求失败 | 显示Toast"操作失败，请检查网络"，停留在当前页面 |
| 表单校验失败 | 显示具体错误提示，高亮错误字段 |
| 数据保存失败 | 显示Toast"保存失败，请稍后重试"，保留用户输入 |
| 图片URL无效 | 显示占位图，提示"图片加载失败" |
| 删除失败 | 显示Toast"删除失败，请稍后重试" |
| 并发编辑冲突 | 后提交的覆盖先提交的（简单策略） |

## 三、总结

### 3.1 核心功能对比

| 功能模块 | 用户价值 | 技术实现 | 优先级 |
|---------|---------|---------|-------|
| 文旅内容浏览 | 获取安徽文旅信息，满足浏览需求 | Supabase数据库 + 点赞收藏评论 | P0（最高） |
| 徽派文化趣味问答 | 趣味化学习徽文化，增强互动性 | 随机题目 + 印章收集机制 | P0（最高） |
| 内容管理（可修改） | 降低内容更新门槛，提高更新频率 | 小程序内表单编辑 + 数据库操作 | P1（高） |

### 3.2 产品亮点

1. **徽派美学设计**：独特的视觉风格，强化安徽文化认同
2. **趣味化学习**：问答+印章机制，寓教于乐
3. **全员可编辑**：降低运营成本，提高内容更新频率
4. **轻量化交互**：无需登录，点赞收藏评论一键操作
